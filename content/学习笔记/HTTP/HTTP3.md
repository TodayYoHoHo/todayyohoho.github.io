# HTTP3

# HTTP/3 主要解决的问题

## 1. 解决 **TCP 队头阻塞（最核心）**

- HTTP/1.1：单连接串行请求，**一个请求卡顿，后面全部堵死**。
- HTTP/2：同一 TCP 连接内多路复用，但**TCP 本身丢包重传依然会阻塞整个连接**（TCP 层队头阻塞）。
- HTTP/3：基于 UDP，**每个流独立传输**，单流丢包只影响自身，其他请求不受影响。

## 2.改变编码格式，解决头部编码队头阻塞问题

HTTP/3 改用 ​**QPACK**：

1. 扩充静态表（61 项 → 98 项），压缩效率更高；
2. 将动态表编解码放到**独立单向 Stream** 传输，和业务流解耦；
3. 彻底规避 HPACK 时序带来的编码阻塞。

依靠「静态表 + 动态表 + 哈夫曼编码」实现头部压缩

## 3. 解决 **TCP 握手、TLS 握手延迟高**

传统流程（HTTP1.1/2）：  
TCP 三次握手 + TLS 四次握手 → **多轮往返（RTT）** 延迟高。

- HTTP/3（QUIC）：**0-RTT / 1-RTT 握手**，首次/重连大幅减少握手次数，**连接建立更快**。
